# 🌐 دليل تشغيل واختبار الدروع الذكية (Enterprise NGFW) على الأجهزة الوهمية (VM)

يوضح هذا الدليل الخطوات العملية خطوة بخطوة لبناء معمل اختبار مبني على الأجهزة الوهمية (Virtual Machines) باستخدام برامج مثل **VMware Workstation** أو **VirtualBox**.

لتجربة جدار الحماية، سنقوم بتصميم شبكة (Man-in-the-Middle) حيث يمر كل ترافيك الشبكة عبر خادم NGFW الذي قمنا ببرمجته.

---

## 📐 1. خريطة الشبكة (Network Topology)

تحتاج إلى إنشاء **3 أجهزة وهمية (VMs)** كالتالي:

```text
       [ جهاز المستخدم / المهاجم ]         [ سيرفر Enterprise NGFW ]          [ شبكة الإنترنت / السيرفر الهدف ]
       (Client / Attacker VM)           (Kali / Ubuntu Server VM)                (WAN / Internet)
         OS: Windows 10 أو Kali            OS: Ubuntu 22.04 أو Kali
         IP: 192.168.10.2                  IP (LAN): 192.168.10.1                 IP (WAN): DHCP/10.0.0.X
         Gateway: 192.168.10.1             IP (WAN): DHCP / 10.0.0.X
                 |                                     |                                     |
                 |---( شبكة دخلية Host-Only )----------|-----( شبكة خارجية Bridged/NAT )-----|
```

---

## 🛠️ 2. إعدادات الأجهزة (VM Settings)

### السيرفر الرئيسي (Enterprise NGFW VM)
هذا هو الجهاز الذي سيُشغل نظام بايثون الخاص بنا.
1. **نظام التشغيل:** يفضل استخدام **Ubuntu Server** أو **Kali Linux**.
2. **الشبكة الأولى (Network Adapter 1):** اجعلها **NAT** أو **Bridged** (هذا المنفذ سيتم تعيينه كـ `WAN` للاتصال بالإنترنت).
3. **الشبكة الثانية (Network Adapter 2):** أضف كرت شبكة جديد واجعله **Host-Only** أو **Internal Network** أو **LAN Segment** (هذا المنفذ سيتم تعيينه كـ `LAN`).

### جهاز المستخدم (Client VM)
هذا هو الجهاز الذي سنقوم باختبار الحظر عليه.
1. **نظام التشغيل:** Windows أو أي Linux.
2. **الشبكة (Network Adapter):** اجعلها **نفس الشبكة الداخلية** التي تم تعيينها للكرت الثاني في سيرفر NGFW (مثلاً: Host-Only أو Internal Network).
3. **إعدادات الـ IP (داخل الويندوز/اللينكس):**
   - أعطه IP يدوي: `192.168.10.2`
   - البوابة الافتراضية (Default Gateway): `192.168.10.1` (IP سيرفر الـ NGFW)
   - خادم الـ DNS: `8.8.8.8` أو `1.1.1.1`

---

## ⚙️ 3. إعداد خادم الـ NGFW (خطوات التشغيل داخل الـ VM)

قم بتشغيل السيرفر (Ubuntu/Kali) واتبع هذه الخطوات بدقة:

### 1. إعداد الـ IP للمنفذ الداخلي (LAN)
افتح الطرفية (Terminal) وضع IP لكارت الشبكة الداخلي (ولنفرض أن اسمه `eth1`):
```bash
# استبدل eth1 باسم كرت الشبكة الداخلي لديك (اكتشفه بـ ip a)
sudo ip addr add 192.168.10.1/24 dev eth1
sudo ip link set eth1 up
```

### 2. تفعيل التوجيه (IP Forwarding) لتمرير الإنترنت
السماح للسيرفر بالعمل كراوتر لتمرير بيانات المستخدمين إلى الإنترنت:
```bash
sudo sysctl -w net.ipv4.ip_forward=1
```

### 3. إعداد الـ NAT (مشاركة الإنترنت)
للسماح للأجهزة الداخلية بتصفح الإنترنت، استخدم كرت الإنترنت (ولنفرض أن اسمه `eth0`):
```bash
sudo iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
```

### 4. تحويل التنبؤات (Redirect) لنظام الكور الخاص بنا
لجعل الترافيك يمر عبر محركات الذكاء الاصطناعي وجدار الحماية الخاص بنا، نُحوّل بورت 80 و 443 إلى بورتات النظام:
```bash
sudo iptables -t nat -A PREROUTING -i eth1 -p tcp --dport 80 -j REDIRECT --to-port 8080
sudo iptables -t nat -A PREROUTING -i eth1 -p tcp --dport 443 -j REDIRECT --to-port 8443
```
*ملاحظة: محرك `eBPF / XDP` الخاص بنا يتم ربطه بالكروت تلقائياً لالتقاط الترافيك بدون الحاجة لـ iptables، لكن هذه القواعد مفيدة لاستقبال المواقع على خوادم البروكسي الداخلية الخاصة بنا.*

## 🔥 4. تشغيل النظام عبر الـ CLI & Web UI

### الخطوة الأولى: تشغيل محرك الكور (Core Engine)
من داخل مجلد المشروع في الـ VM الأساسي:
```bash
# يجب تشغيل النظام بصلاحيات Root لكي تعمل الـ eBPF
sudo python3 main.py
```
*(ستظهر لك شاشة الترحيب الخضراء توضح أن نظام الفحص وسيرفر API قد بدآ العمل).*

### الخطوة الثانية: تخصيص المنافذ (Interfaces Map)
افتح شاشة طرفية (Terminal) أخرى لاستخدام الـ CLI، لتوزيع كروت الشبكة التي تم اكتشافها:
```bash
# تحديد كرت الإنترنت كـ WAN
sudo ngfw interfaces assign eth0 --role WAN

# تحديد كرت الشبكة الداخلية كـ LAN
sudo ngfw interfaces assign eth1 --role LAN
```

### الخطوة الثالثة: تشغيل لوحةควบคุม React Dashboard
في نافذة طرفية أخرى:
```bash
cd web-ui
npm install
npm run dev
```

## 🧪 5. سيناريوهات الاختبار الحية (How to Test)

بمجرد عمل النظام، انتقل لجهاز **المستخدم (Client VM)** وقم بتنفيذ الاختبارات الآتية للتأكد من فاعلية NGFW:

### الاختبار الأول: فحص فك تشفير المواقع (SSL Inspection)
1. افتح متصفح واذهب إلى `https://google.com`.
2. انقر على أيقونة **"القفل"** الموجودة في شريط الرابط لمعرفة من الذي أصدر شهادة الأمان.
3. **النتيجة المتوقعة:** يجب أن تجد أن الشهادة مصدرة من `Enterprise NGFW Root CA` وليس من `Google Trust Services`. هذا يثبت نجاح اعتراضنا للبيانات وتطبيق الـ SSL Inspection. *(قد تحتاج لتثبيت شهادتنا `root-ca.crt` في الويندوز لتخطي شريط التحذير الأحمر).*

### الاختبار الثاني: محرك الـ Web Filtering
1. قم بإنشاء قاعدة لمنع تصفح مواقع التواصل من الـ Dashboard أو عبر الـ CLI (مثلاً حجب `facebook.com`).
2. حاول زيارة `facebook.com` من جهاز الـ Client.
3. **النتيجة المتوقعة:** لن تفتح الصفحة، وستظهر شاشة حجب الخاصة بجدار الحماية (Block Page).

### الاختبار الثالث: الذكاء الاصطناعي وهجمات الـ DoS
1. قم بفتح لوحة تحكم React وانتقل إلى صفحة **Live Traffic** لمراقبة الخطط.
2. من جهاز الـ Client، حمل أداة مثل `hping3` أو قم بعمل Ping سريع مكثف.
   `hping3 --flood 192.168.10.1`
3. **النتيجة المتوقعة:** سيبدأ الـ **Anomaly Detector** بتسجيل سلوك شاذ، ثم يقوم نظام **XDP/eBPF kernel** بحظر الـ IP `192.168.10.2` الخاص بجهاز العميل فوراً من خلال النواة وسينقطع اتصاله بالإنترنت.
