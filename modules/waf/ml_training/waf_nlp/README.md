# WAF NLP Model (Natural Language Processing)

## 📌 الوصف السريع (Overview)
هذا النموذج عبارة عن شبكة عصبية لغوية تستخدم تقنيات البرمجة اللغوية العصبية (NLP) للتمييز بين النصوص السليمة الواردة من المستخدمين، وبين الحمولات الخبيثة (Malicious Payloads) التي تُستخدم في هجمات حقن قواعد البيانات (SQLi)، البرمجة عبر المواقع (XSS)، واختراق المسارات (Path Traversal).
يعتمد هذا النموذج على تحليل **سياق** وترتيب الكلمات، مما يجعله قادراً على اكتشاف هجمات الـ Zero-day التي لم تُبرمج كقواعد Regex صريحة.

## 🧠 بنية النموذج (Architecture)
يستخدم النموذج تقنيات **TF-IDF Vectorization** (أو تضمين الكلمات المعجمي Embedding) لتحويل الكلمات والنصوص الواردة في الطلبات (Requests) إلى مصفوفات رياضية، وموصول بشبكة تصنيف تصدر قراراً (Score) ما بين `0.0` (آمن جداً) إلى `1.0` (خبيث جداً).

## 🛠 كيف يتم التدريب؟ (Training Process)
النموذج هو نموذج **مُدرَّب مُسبقاً (Pre-trained Offline)**، أي أنه يجب تدريبه قبل إطلاق الجدار الناري للإنتاج باستخدام مجموعة بيانات تحتوي على آلاف الهجمات المعروفة والاستخدامات الجيدة، وحفظ النتيجة في ملف ليقرأه הـ WAF بسرعة شديدة في الذاكرة:

### ملفات التدريب:
- يجب أن تكون البيانات مجهزة كملفات نصوص أو قوائم CSV توضع في مجلد `datasets/` داخل هذا المسار.
- ينقسم الـ Dataset إلى `X_train` (النصوص) و `y_train` (العلامات `1` للهجوم، `0` للسليم).

### أوامر تشغيل التدريب:
لتدريب النموذج أو تحديثه بعد إضافة قواعد جديدة:
```bash
cd M:\نسخ المشروع\enterprise_ngfw
python -m modules.waf.ml_training.waf_nlp.train
```

سيتم حفظ النموذج النهائي ذو الوزن الأمثل في ملف (مثل `waf_nlp_model.pt` أو `model.pkl`) في نفس مساره، وسيقوم جدار الحماية (الـ `waf_inspector.py`) عند التشغيل التالي بتحميل النسخة الجديدة مباشرة.
