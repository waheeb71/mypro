# Bot Detection Model (اكتشاف البوتات الجيل الجديد)

## 📌 الوصف السريع (Overview)
محرك اكتشاف البوتات هو نموذج التعلم الآلي المسؤول عن التفريق بين المستخدمين الفعليين (البشر، المتصفحات الشرعية) وبين السكربتات المبرمجة آلياً (البوتات، Scrapers، وأدوات الهجوم العشوائي الأوتوماتيكية).
لا ينخدع هذا النموذج عن طريق تزوير المستخدم الآلي لاسم متصفحه (User-Agent)، لأنه يعتمد بدلاً من ذلك على **السلوك (Behavioral Analysis)** والمعالم الهيكلية للطلبات الخاصة بـ HTTP.

## 🧠 البنية والمعالم المستخدمة (Features & Architecture)
يعتمد النموذج على خوارزميات التصنيف (مثل Random Forest أو MLP Classifier) ويتغذى على عدة خصائص (Features) تُستخرج من كل طلب (Request)، منها:
1. **سرعة وكثافة الطلبات:** معدل زيارات الـ IP الواحد والمستخدم.
2. **غياب ميزات المتصفح:** هل توجد هيدرز (Headers) اعتيادية مثل `Accept-Language` و `Accept-Encoding` وغيرها.
3. **تطابق JA3:** فحص بصمة الـ TLS للتحقق من هوية عميل الـ SSL الفعلي.

## 🛠 كيف يتم التدريب؟ (Training Process)
يتم تدريب هذا النموذج **يدويًا (Offline)** لكي يتعرف على أشكال أدوات الانحرافات الأمنية والمكتبات البرمجية الشهيرة (مثل `Python-requests`, `curl`, `Selenium`).

### ملفات التدريب:
تعتمد على سجلات دخول (Access Logs) مُعنونة بعلامات مازّة بين طلبات طبيعية وطلبات تم إنشاؤها عبر سكريبتات. يحتاج لملف CSV ضمن مجلد `datasets/` يحوي أعمدة للـ Features.

### أوامر تشغيل التدريب:
لإعادة بناء واختبار وتدريب نموذج استكشاف البوتات:
```bash
cd M:\نسخ المشروع\enterprise_ngfw
python -m modules.waf.ml_training.bot_detection.train
```

سيقوم النظام بإعادة بناء شجرة القرار (أو الأوزان)، توليد تقرير بنسبة الدقة، ثم يقوم بحفظ الملف ليكون جاهزاً للـ WAF ليستخدمه في الكشف اللحظي.
